Diabetes risk screening with a recall-first neural network.

UdaciHealth is a PyTorch-based screening model trained on CDC Behavioral Risk Factor Surveillance System data. The project starts with a balanced instructional dataset, then extends the work with threshold tuning, feature engineering, and class-weighted retraining on the original imbalanced population.

What this project solves

The model acts as a pre-screening layer for a hospital network that cannot send every patient to full diabetes testing. In that setting, missing a true positive is the costlier error, so the work optimizes for screening recall rather than raw accuracy.

Core approach

The final workflow uses a stratified train/validation/test split, standardized features, a PyTorch multi-layer perceptron, validation-driven threshold tuning, and a class-weighted retrain on the original population distribution.

Stand-out extensions

Threshold tuning Feature engineering Class-weighted training Original imbalanced CDC data Static notebook publishing

The base rubric is fully completed, but the project also implements the stronger portfolio work that matters in practice: the screening threshold is tuned on validation data, a small set of clinically motivated interaction features is tested, and the model is re-evaluated on the original imbalanced CDC population instead of stopping at the balanced teaching subset.

Run locally

Create a virtual environment, install `requirements.txt`, and open the notebook from the repository root. The repo includes both the balanced teaching dataset and the original imbalanced CDC file used for the extension experiments.

`python -m venv .venv` `python -m pip install -r requirements.txt` `jupyter notebook starter-kit/diabetes_prediction_mlp.ipynb`
